## Approach

Create a new `commands/merge.sh` shell script following the same `include_str!` pattern
already established by `commands/draft_pr.sh`. The script will run `gh pr checks
$FORZA_PR_NUMBER --watch` before `gh pr merge`, so the merge only proceeds if all CI
checks pass. In `stage.rs`, add a `MERGE_COMMAND` constant loaded with `include_str!`
from the new script, then replace the four inline `"gh pr merge $FORZA_PR_NUMBER
--squash --delete-branch"` strings in the bug, feature, chore, and pr-merge workflow
definitions with `MERGE_COMMAND`. This keeps the merge command in a single auditable
location and makes it user-overridable via the `command` field in `forza.toml`, exactly
as the draft-PR command is today.
