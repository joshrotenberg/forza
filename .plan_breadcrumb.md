## Approach

Add a `stage_name: &str` parameter to the `AgentExecutor::execute` trait method in `crates/forza-core/src/traits.rs`. The `pipeline.rs` loop already binds `let stage_name = stage.kind.name()` before calling `agent.execute()`; pass that value as the new argument. In `adapters.rs`, receive `stage_name` and use it to construct the `PlannedStage` with the correct `StageKind` instead of hardcoding `StageKind::Implement`. Add a `from_name` helper method to the old forza crate's `StageKind` (in `crates/forza/src/workflow.rs`) that maps the canonical snake_case name back to the enum variant, defaulting to `Implement` for any unknown name. This fixes the executor-level log line (`stage="implement"` for all agent stages) and ensures tool-scoping in `ClaudeAdapter::execute_stage` also uses the correct stage kind. The test mock `SuccessAgent` in `pipeline.rs` must also be updated to accept the new parameter.
