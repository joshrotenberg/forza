# Plan: Fix merge stage worktree conflict

## Files to modify
- `crates/forza-core/src/commands/merge.sh`

## Approach

The merge stage command `gh pr merge --squash --delete-branch` fails when run inside a worktree because `--delete-branch` causes `gh` to try switching the local checkout back to `main` after deleting the branch — but `main` is already checked out in the parent worktree, producing `fatal: 'main' is already used by worktree at '...'`.

The fix is to remove the `--delete-branch` flag from `merge.sh`. Branch cleanup after merge can be handled by GitHub's repository-level "Automatically delete head branches" setting, which is the standard approach and avoids any git worktree conflicts. The `pr-merge` workflow already avoids this problem by setting `needs_worktree: false`, but bug/feature/chore workflows run the merge stage inside a worktree context, so the flag is unconditionally harmful there.
