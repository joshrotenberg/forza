# Plan: per-agent model defaults and budget guardrails (closes #531)

## Files to modify

- crates/forza/src/config.rs
- crates/forza/src/runner.rs
- crates/forza/src/adapters.rs
- crates/forza/src/executor.rs
- crates/forza-core/src/pipeline.rs
- crates/forza-core/src/traits.rs
- crates/forza-core/src/testing.rs

## Approach

Add a `PerAgentConfig` struct with `model: Option<String>` and `max_budget_usd: Option<f64>`, then add an `agents: HashMap<String, PerAgentConfig>` table to `RunnerConfig` so users can write `[agents.claude]` and `[agents.codex]` blocks in `forza.toml`.

Update `effective_model()` to take the active agent name and resolve `route.model > agents[agent].model > global.model`, eliminating the need for the adapter-level model-prefix filtering workaround added in #525 (though the filter can stay as a safety net).

Add `max_budget_usd: Option<f64>` to `PipelineConfig` and to the `AgentExecutor::execute()` trait signature (alongside the existing `model` parameter), then wire it through `ClaudeAdapter` → `QueryCommand::max_budget_usd()` in executor.rs and through the matching `CodexAgentAdapter` path. Also add `max_budget_usd: Option<f64>` to `Route` for per-route overrides, with resolution order `route > agents[agent] > global (max_cost_per_issue as fallback)`. Update `MockAgent` in testing.rs to match the new trait signature.
