# Plan: check agent CLI version at startup

## Files to modify

- crates/forza/src/deps.rs

## Approach

The `claude-wrapper` crate (v0.4.1) already exposes `Claude::cli_version()` and `Claude::check_version(&CliVersion)` with `CliVersion::new(major, minor, patch)`. The `check_agent()` function in `deps.rs` currently only verifies the agent binary exists (exit code from `--version`). Enhance `check_agent()` to, when the agent is `"claude"`, build a `Claude` client and call `check_version()` against a minimum version constant (e.g., `2.0.0`). On version mismatch, emit a warning via `eprintln!` rather than a hard error so forza remains usable with slightly older CLIs. Also update `doctor_agent()` to surface the version comparison result. For `"codex"` or unknown agents, keep the existing behavior (presence check only) since codex-wrapper does not expose version checking.
