You are generating a `forza.toml` configuration file for the repository `{repo}`.

Inspect the repository and write a well-commented `forza.toml` to `{output}`.

## Steps

1. **Detect language and tooling**
   - Check for `Cargo.toml` (Rust), `package.json` (Node/JS), `pyproject.toml` / `setup.py` (Python), `go.mod` (Go), etc.
   - Choose appropriate `[validation].commands` based on what you find.

2. **Inspect CI workflows**
   - Read `.github/workflows/*.yml` to understand what lint, test, and build commands are used.
   - Use these to inform `[validation].commands`.

3. **Query existing GitHub labels**
   - Run `gh label list --repo {repo} --limit 50 --json name` to list existing labels.
   - Map relevant labels (e.g. `bug`, `enhancement`, `feature`) to routes.

4. **Check branch protection**
   - Run `gh api repos/{repo}/branches/main/protection` (ignore errors if protection is not enabled).
   - If required reviews or status checks are present, set `authorization_level = "contributor"` (do not auto-merge).
   - Otherwise `authorization_level = "trusted"` is acceptable.

5. **Detect default branch**
   - Run `git symbolic-ref refs/remotes/origin/HEAD 2>/dev/null | sed 's|refs/remotes/origin/||'`
   - Use this to set `branch_pattern` base if it's not `main`.

6. **Write the config**
   - Write the resulting TOML to `{output}`.
   - Add a comment above each section explaining its purpose.
   - Include at minimum: `[global]`, `[security]`, `[validation]`, and at least one route.

7. **Optionally generate GitHub Action workflow**
   - If `.github/workflows/` exists, also write `.github/workflows/forza.yml` with a basic forza action setup.
   - The workflow should trigger on `issues.labeled`, `check_suite.completed`, and `workflow_dispatch`.

## Output format

The config file must be valid TOML parseable as a forza `RunnerConfig`. A minimal example:

```toml
# forza.toml — generated by `forza init --auto`

[global]
model = "claude-sonnet-4-6"
gate_label = "forza:ready"
branch_pattern = "automation/{issue}-{slug}"

[security]
authorization_level = "contributor"

[validation]
commands = [
    "cargo fmt --all -- --check",
    "cargo clippy --all-targets -- -D warnings",
    "cargo test",
]

[repos."{repo}".routes.bugfix]
type = "issue"
label = "bug"
workflow = "quick"

[repos."{repo}".routes.features]
type = "issue"
label = "enhancement"
workflow = "feature"
```

## Built-in workflows

| Name | Stages | Use for |
|------|--------|---------|
| `quick` | implement, test, open_pr | Bugs, chores, small tasks |
| `feature` | plan, implement, test, review, open_pr | Larger features |
| `research` | research, comment | Investigation, no code changes |
| `pr-fix` | revise_pr, fix_ci | Fix PR review feedback + CI |
| `pr-rebase` | revise_pr | Rebase a PR |

Legacy aliases: `bug` and `chore` map to `quick`.

Do NOT commit any files. Write only `{output}` (and optionally the action workflow).
