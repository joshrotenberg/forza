## Review: issue #531

### Summary

Per-agent model defaults and budget guardrails. Adds `[agents.<name>]` config blocks with `model` and `max_budget_usd` fields. Resolution order: route > agents[agent] > global for both model and budget.

### Issues found

| Severity | File | Line | Description |
|----------|------|------|-------------|
| medium | crates/forza/src/config.rs | ~780 | `effective_max_budget_usd()` exists and is tested but never called in production code. `runner.rs` duplicates the same resolution logic inline instead of calling this method. Dead code suggests incomplete refactoring. |
| low | crates/forza/src/adapters.rs | CodexAgentAdapter::execute | Codex logs a warning when `max_budget_usd` is set but silently ignores it. Users setting budget guardrails may assume they apply to all agents. |
| low | crates/forza/src/config.rs | PerAgentConfig | No integration test verifying `max_budget_usd` threads through the pipeline to the Claude adapter — only unit tests on resolution logic. |

### Detail: duplicate budget resolution logic

`runner.rs` resolves budget inline:

```rust
let max_budget_usd = work
    .route
    .max_budget_usd
    .or_else(|| config.agents.get(agent_name).and_then(|a| a.max_budget_usd))
    .or(config.global.max_cost_per_issue);
```

`config.rs` defines `effective_max_budget_usd()` with identical logic that is never called outside tests. These should be consolidated — runner.rs should call the method.

### Strengths

- Three focused unit tests covering the full resolution chain (route > agents > global)
- All call sites properly updated to pass agent name
- Backward-compatible: all new fields are `Option`
- `effective_model()` refactored correctly and called from all dispatch paths (main, api, mcp)
- Doc comment escaping (`agents\[agent\]`) handled correctly

### Verdict: PASS

No high-severity issues. The medium-severity issue (dead code / duplicate logic) is a code quality concern but does not affect correctness — the inline logic in `runner.rs` is equivalent to the untested method. Budget resolution and model resolution work as specified. Tests cover the priority chain.
